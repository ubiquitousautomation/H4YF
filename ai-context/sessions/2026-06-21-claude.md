## Decisions Made
- **Sync tool**: rclone (headless Linux container, no GUI)
- **Source of truth**: git (`ai-context/`) with Drive as secondary sync layer
- **Push safety**: rclone copy (never deletes) for default push; rclone sync reserved for explicit fullsync
- **Session logs**: tracked in git so Claude can read Gemini's history; opt-out documented in .gitignore
- **Conflict resolution**: sequential work assumed; human decides on true conflicts; status command available
- **PostToolUse auto-commit**: deferred — manual pull workflow is sufficient for now
- **Dry-run testing**: deferred until Drive auth is configured

## Observations / Learnings
- Claude Code Stop hooks run from the project root — no need for absolute paths
- `rclone bisync` errors on first run without `--first-run` flag — auto-detection added
- `rclone config create ... --all` prompts for every option interactively — wrong for scripted setup
- The biggest risk in the sync design is destructive push overwriting Drive-side Gemini edits — mitigated by switching to rclone copy
